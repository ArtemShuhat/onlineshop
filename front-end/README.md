# Архитектура фронтенда

Этот проект использует **Feature-Sliced Design (FSD)**. Каждый слой хранит только свою ответственность и экспортирует публичное API через `index.ts`.

## Слои

- `app` — точка входа Next.js, глобальные провайдеры, layouts. Не содержит бизнес-логики.
- `processes` — длинные пользовательские сценарии (например, checkout). Собирают несколько features и widgets.
- `pages` — реализация маршрутов. Хранят описание страницы и подключают необходимые widgets/processes.
- `widgets` — крупные составные блоки интерфейса (header, hero, product-grid). Внутри используют entities и features.
- `features` — завершённые пользовательские возможности: формы входа, добавление в корзину, фильтры и т.п. Имеют собственные `ui`, `model`, `lib`.
- `entities` — бизнес-сущности: `user`, `product`, `category`, `order`. Содержат модели и UI-примитивы вокруг конкретной сущности.
- `shared` — общие ресурсы: дизайн-система, api-клиент, конфигурация, утилиты, типы без бизнес-контекста.

## Правила

1. Импорты между слоями идут только через публичное API (`index.ts`).
2. Нельзя опускаться «вниз» по структуре: `pages` не лезет напрямую в `features`, а использует `widgets` / `processes`.
3. В `shared` не допускается бизнес-логика. Только общие инструменты.
4. Новая функциональность описывается в README конкретного сегмента (`entities/product/README.md` и т.д.).
5. При перемещении кода обновляйте Storybook/тесты.
